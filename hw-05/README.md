<h1>Домашнее задание №5</h1>
<h3>Текущее задание:</h3>
<p>Разработать однопользовательское консольное приложение позволяющее вести каталог книг в библиотеке.
Каждая книга имеет автора и как минимум один жанр</p>
<h4>Требования к реализации:</h4>
<ul>
<li>Использовать Spring JDBC и реляционную базу H2</li>
<li>Для параметризованных запросов использовать NamedParametersJdbcTemplate</li>
<li>Каждая сущность должна храниться в своей таблице</li>
<li>Отношение между книгой и автором - многие-к-одному
(у книги один автор, но у автора может быть несколько книг)</li>
<li>Отношение между книгой и жанром многие-ко-многим (у книги много жанров, и у жанра может быть несколько книг)</li>
<li>Интерфейс выполняется на Spring Shell. Для сущности книги, предусмотреть операции создания, обновления,
удаления, вывода всех книг, а также вывода книги по id</li>
<li>Предусмотреть операции вывода полного списка сущностей авторов и жанров</li>
<li>Не делать абстрактных или обобщенных сущностей</li>
<li>Не делать абстрактных или обобщенных Dao</li>
<li>Не делать двунаправленных связей (в книге автор, в авторе книги)</li>
<li>Проблема N+1 должна быть решена</li>
<li>С помощью @JdbcTest сделать интеграционные тесты всех методов дао жанров (со встроенной БД)</li>
<li>Создание и инициализация схемы БД сделать через систему миграций (Liquibase/Flyway)</li>
</ul>
